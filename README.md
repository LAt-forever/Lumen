# Lumen

Lumen 是一个本地优先的个人 AI 知识库和长期记忆助手。当前版本是 Phase 1.6 原型，重点是在已有记忆基础上添加可解释的关系图谱：

```text
添加资料
-> 建立索引
-> 基于资料提问并查看引用
-> 从对话中提取待确认记忆
-> 确认、编辑、合并或遗忘记忆
-> 用标签、收藏和全局搜索找回资料、记忆和回答
-> 在今日回顾里查看最近变化和下一步建议
-> 在状态页查看运行健康并修复失败资料
-> 在记忆图谱里查看记忆关系和来源
```

## 当前能力

- 中文工作台界面。
- 主要界面、按钮、占位文案、空状态、状态标签和后端兜底回答已中文化。
- 添加纯文本资料并自动索引。
- 上传 `.txt`、`.md` 和可提取文本的 `.pdf` 文件作为资料。
- 捕获网页链接并提取安全 HTML 文本；抓取失败会留下失败状态和错误信息。
- 基于已索引资料提问，回答会显示引用片段和置信度。
- 搜索结果和回答引用会显示匹配原因，例如命中的关键词或日期。
- 全局搜索可同时检索资料片段、资料记录、已确认记忆和对话消息/回答。
- 全局搜索支持按结果类型、标签和收藏状态筛选，并显示结果类型、匹配原因、相关度、标签和收藏状态。
- 可为资料、记忆和对话回答收藏，方便后续找回。
- 可查看本地生成的标签建议，并在确认后把建议转为正式标签；也可以忽略建议。
- 默认使用摘录模式回答，不需要模型 API key。
- 可通过环境变量启用 OpenAI-compatible LLM 回答，并要求回答基于已检索资料和已确认记忆。
- 可在设置页创建、编辑、测试和启用 OpenAI-compatible 模型配置；配置会保存在本机 SQLite。
- 运行时优先使用已启用的 SQLite 模型配置；没有启用配置时回退到环境变量。
- 模型 API key 只在本机保存，读取接口只返回是否已配置，不返回明文。
- 提问优先使用 `/api/chat/stream` 的 SSE 流式接口，前端会边接收边显示回答；失败时回退到非流式接口。
- LLM 未配置、请求失败或证据不足时会回退到摘录模式，并在界面显示回退说明。
- 针对中文日期问题做聚焦引用，例如询问 `2026年6月1日做了什么工作？` 时优先返回同一天的内容片段。
- 从对话中提取候选记忆，例如项目、偏好、目标。
- 在记忆收件箱中确认或忽略候选记忆。
- 在记忆页编辑、遗忘和合并已确认记忆。
- 在记忆页查看记忆来源，并看到保守的重复记忆合并建议。
- 可在记忆之间创建有向关系，关系类型包括相关、属于、导致、支持、矛盾和合并入。
- 合并记忆时会自动创建「合并入」关系，保留来源记忆的血统。
- 可在重复记忆建议旁点击「标记为相关」，将其转为正式的 `related_to` 关系。
- 新增「图谱」视图，以单条记忆为中心展示关系子图。
- 图谱支持调整探索深度（1-3 层），点击节点可重新以该记忆为中心。
- 图谱使用自定义 SVG 力导向布局，不依赖外部可视化库。
- 关系显示来源（provenance）和强度（strength），遗忘的记忆和关系不会出现在图谱中。
- 左侧导航已能切换今天、提问、资料库、记忆、图谱、搜索、回顾、状态和设置视图。
- 资料库可查看资料详情、索引片段数、解析错误，并删除不再需要的资料。
- 状态页会展示模型运行来源、资料索引概况、失败资料、待确认标签建议和修复动作。
- 失败链接资料可以从状态页或资料相关入口重试抓取和索引。
- 今日回顾会展示资料、记忆和建议动作。
- 设置页会显示安全的模型运行诊断，例如 LLM 模式缺少模型名或 API key 的提示。

## 当前限制

- 现在是本地原型，不是完整生产应用。
- 默认回答模式仍是摘录式，不需要模型 API key；LLM 总结是可选能力。
- Phase 1.5 的模型配置仍只支持 OpenAI-compatible 聊天接口。
- SQLite 中保存的 API key 是本机明文存储，依赖本机文件权限保护；暂未接入系统 keychain 或加密密钥库。
- PDF 只支持 selectable-text 文本提取，扫描件 OCR 暂未接入。
- 网页抓取只处理提交的单个 URL；登录页、JavaScript 重渲染页面和递归爬取暂未实现。
- 图片知识库、Agent 配置 UI、外部 reranker 配置和完整任务队列面板仍未实现。
- 标签建议是本地确定性启发式建议，不会默认调用 LLM 自动打标签。
- 后端 API 的置信度字段仍使用内部枚举，例如 `grounded`、`memory-only`、`weak`；前端会显示为中文。
- 数据默认写入后端目录下的本地 SQLite 数据库 `backend/lumen.db`。

## Phase 1.7: Ingestion Expansion

- **PDF OCR**: Scanned/image PDFs are automatically OCR'd using Tesseract
- **Image Knowledge Base**: Upload images (PNG, JPG, GIF, WebP) — text is extracted via OCR and described via AI vision
- **DOCX & EPUB Support**: Import Word documents and e-books
- **Advanced Web Scraping**: Recursive crawling with Playwright (JS-rendered pages, configurable depth)
- **Bookmark Import**: Import browser bookmarks (Netscape HTML format)
- **Batch Upload**: Select and upload multiple files at once

## 可选 LLM 回答

默认不需要模型配置：

```bash
LUMEN_LLM_MODE=extractive
```

如果要启用 OpenAI-compatible 聊天模型，推荐在「设置」视图创建模型配置，填写 Base URL、模型名和 API key，然后点击「测试连接」和「设为当前」。

也可以在启动后端前使用环境变量作为 fallback 配置：

```bash
export LUMEN_LLM_MODE=llm
export LUMEN_LLM_PROVIDER=openai-compatible
export LUMEN_LLM_BASE_URL=https://api.openai.com/v1
export LUMEN_LLM_MODEL=<model-name>
export LUMEN_LLM_API_KEY=<api-key>
export LUMEN_LLM_FALLBACK_ENABLED=true
```

当存在已启用的 SQLite 模型配置时，Lumen 会优先使用它；否则使用环境变量。API key 不会通过读取接口返回。模型请求失败或证据不足时，回答会回退到摘录模式。

## 使用流程

1. 在中间的「询问或记录」输入框粘贴一段资料。
2. 点击「添加资料」，资料会保存并建立索引。
3. 也可以切到「文件」上传 `.txt`、`.md` 或 `.pdf`，或切到「链接」添加网页 URL。
4. 在同一个输入框输入问题，点击「询问 Lumen」。
5. 在「对话」里查看流式生成的回答。
6. 在右侧「当前上下文」里查看置信度和引用片段。
7. 查看引用片段下方的匹配原因，确认 Lumen 为什么选中这条资料。
8. 如果右侧「记忆收件箱」出现候选记忆，点击「确认」或「忽略」。
9. 进入「记忆」视图，编辑、遗忘或合并已确认记忆，并查看可能重复的记忆建议。
10. 进入「资料库」视图，查看资料详情或删除不再需要的资料。
11. 进入「搜索」视图，全局检索资料、记忆和对话回答，并用类型、标签或收藏过滤。
12. 在资料、记忆或回答旁点击收藏，或确认/忽略标签建议。
13. 进入「状态」视图，查看运行状态、失败资料和待确认标签建议，并重试失败资料。
14. 在「今日回顾」里查看建议动作。
15. 进入「图谱」视图，选择中心记忆并查看其关系子图；可调整探索深度（1-3 层），点击节点可重新以该记忆为中心。
16. 在记忆详情或重复建议旁点击「添加关系」，选择目标记忆、关系类型、来源和强度，在两条记忆之间建立有向关系。
17. 在重复记忆建议旁点击「标记为相关」，将建议转为正式的 `related_to` 关系。
18. 进入「设置」视图，管理本机模型配置并查看当前运行来源。

可以用下面的资料做烟测：

```text
Lumen 是一个本地优先的个人 AI 知识库，用来保存资料、检索知识，并通过确认机制记住用户偏好。
```

添加资料后提问：

```text
Lumen 是什么？
```

也可以添加工作日报后提问：

```text
2026年6月1日，我完成了 Lumen 中文化界面、日期检索和记忆收件箱烟测。
```

```text
2026年6月1日做了什么工作？
```

触发记忆候选可以输入：

```text
我喜欢用中文记录项目目标，Lumen 项目的目标是做成本地优先的个人知识库。
```

期望「记忆收件箱」出现候选记忆，可以点击「确认」或「忽略」。

确认后进入「记忆」视图，可以修改记忆内容、将临时记忆遗忘，或把两条相关记忆合并。


1. 启动后端和前端。
2. 打开 `http://127.0.0.1:5173/`。
3. 添加一条中文资料。
4. 提问并确认回答有中文内容、引用片段和置信度。
5. 输入一句包含「项目」「喜欢」「目标」等关键词的话，确认记忆收件箱出现候选记忆。
6. 点击「确认」或「忽略」，确认界面刷新正常。
